Two proposals for the attachment edge

A. Nearest edge (dynamic)

The line picks whichever box edge faces the source endpoint. Feels natural when the source is far off to one side, but can look odd when the source is roughly opposite the context (line approaches the box from the wrong side, e.g., enters the top of the ACL even though the ACL semantically guards the context to its right).

B. Opposite from context (locked)

The line always lands on the outer-facing edge of the box (the side away from the parent context). The ACL/OHS reads as a gate the line passes through on its way into the context, regardless of where the source sits. Trade-off: when the source is well off to a perpendicular side, the line bends more than strictly necessary.

Curve style options

Auto bezier

Today's behavior. Control points use a horizontal-vs-vertical bias from endpoint deltas, ignoring which edge of the box is being attached to. Causes the awkward near-vertical entry when source and box are roughly diagonal.

Tangent at edge (recommended)

The curve enters the box perpendicular to the attachment edge (e.g., horizontal for a left-edge attachment). Preserves the organic bezier feel while removing the wrong-angle entry. Control-point length scales with endpoint distance.

Orthogonal

A right-angle elbow: short stub out from the box along its attachment normal, then 90° turns to the source. Reads cleanly but breaks the hand-drawn aesthetic the project has intentionally preserved on context-to-context edges.

Shared rules (both proposals)

  1. Box hugs the context. The ACL/OHS indicator sits flush against the context edge (~2px visual gap so borders read as separate). No relationship line is drawn between the indicator and the context.
  2. Centered on the chosen edge. Attachment sits at the midpoint of the indicator's outer edge.
  3. Same offset as context boxes. Visible path is shortened by the existing context-box offset so the tip sits in a small gap, not on the border.
  4. Arrow head (where applicable). Sits in the gap with its tip pointing at the edge midpoint.
  5. Applies to both ACL and OHS, on all four sides of the parent context.
