Connection UX redesign mockup

Three changes proposed for the DDD Europe 2026 workshop audience: discoverable hover stubs, whole-shape drop targets, and bezier curves preserved. Open connection-ux-research.md for the survey and rationale. Settled in #21: joint anchor side selection (see PR #21).

Panel 1

Hover stub variants

Hover either card. Drag from any stub to start a connection (in the real app -- here the cursor preview shows where the connection would originate).

Arrow nubs: small directional triangles on 4 sides, draw.io's signature affordance. Highest discoverability for non-power-users.
Plus icons: Excalidraw-style; clearer that something is clickable, but loses directional intent and feels noisier.
Perimeter ring: subtle, no discrete handles; whole edge is draggable. Lowest visual cost, lowest discoverability for a 2-hour workshop.
Current: two tiny fixed dots on left/right; the displayed edge floats, so creation and display geometry disagree.

Panel 3

Routing: bezier vs. orthogonal

Drag any card. The rendered edge re-routes in real time. Bezier reads as "flow" and fits the hand-drawn strategic-design register. Orthogonal reads as "system diagram" -- correct for engineering schematics, off-register for context maps.

The "edge wanders" symptom that motivated reconsidering bezier was the per-end independent anchor selection bug fixed in #21. Once anchors are chosen jointly across all 16 side pairs with a facing-side preference, bezier no longer loops out to the wrong side. The remaining "feels janky" lives in the creation affordance (Panel 1) and drop precision (Panel 2), not in the path type.
