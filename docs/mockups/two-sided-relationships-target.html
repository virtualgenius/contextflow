Target behavior

Inspector picker + canvas gestures

Canvas gestures:
· Hover a context for arrow stubs, drag a stub to create a U/D relationship
· Drag a context body onto another to create Shared Kernel 6
· Double-click an arrow to flip its direction 5
· Drag SK contexts apart to auto-convert to Partnership 7
The canvas renders the same way the real React Flow canvas does. Try the gestures.
Behavior summary

What each gesture produces in the data model

Drag arrow stub from A to B → creates a relationship with fromContextId: A, toContextId: B, no pattern, no roles. Renders as a U/D arrow pointing at B (the upstream).

Click Partnership pill → pattern: 'partnership', per-side roles cleared. Renders as a mutual line, no arrow. Peer badges on both contexts.

Click Customer-Supplier pill → pattern: 'customer-supplier', per-side roles cleared. Renders as a U/D arrow with the pattern label.

Click a per-side role pill (OHS, Published Language, Conformist, ACL) → role set, any standalone pattern cleared. Role badge appears on the corresponding context end.

Click the active pattern or role pill again → toggles it off.

Click the arrow in the picker mini-diagram (η) → swaps fromContextId ↔ toContextId; clears per-side roles (since OHS-on-Billing doesn't transfer to OHS-on-Orders).

Double-click the arrow on the canvas (ε) → same as above. Calls the existing swapRelationshipDirection store action.

Drag context body onto another → if no relationship exists between them, creates Shared Kernel. If a relationship exists, prompts to convert.

Drag Shared Kernel contexts apart → pattern auto-switches to 'partnership'. The Mutually Dependent influence is preserved; the user can still pick something else in the picker.

Click Remove relationship → confirms then deletes the Relationship object. Free relationships have no object; documentation of intentional non-integration goes in context notes.
