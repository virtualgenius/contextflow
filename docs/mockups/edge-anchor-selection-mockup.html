Edge anchor selection: current vs proposed

Drag the boxes (and resize from the corner). Both panels stay in sync so you can compare where each algorithm attaches the edge.
Current algorithm picks each end's anchor side independently: it draws a line from each box's center to the other's center, and picks the side that line exits. Proposed algorithm enumerates all 16 (sourceSide, targetSide) combinations and picks the pair with the shortest distance between side midpoints. Try the "Dome Safety / Central Control" preset to see the bug; the two algorithms agree in the side-by-side and stacked presets.

Current: independent side selection

Each end picks its own side from the center-to-center direction. Produces wrap-around curves when boxes have different aspect ratios.

Proposed: joint side selection

Enumerate all 16 (sourceSide, targetSide) pairs; pick the pair with the shortest distance between side midpoints. Anchors stay on facing sides.
