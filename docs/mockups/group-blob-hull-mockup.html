Group Blob: Stroke-Dilated Convex Hull

Verifies the replacement algorithm for the app's group blob rendering before implementation, using variable-size boxes like Flow View context nodes. The blob body is the convex hull of all member rectangle corners, rendered with a fill plus a fat round-join stroke (stroke-width = 2 × padding). The dashed border is a second, explicit path: the rounded hull outline built from offset edges joined by circular arcs at each hull vertex. The thin gray rectangle is the computed bounds (hull extent + padding), the contract that feeds blob positioning.

Members: Padding 60
Try it: drag the boxes. The blob, dashed outline, and bounds all recompute live. Things to verify: padding stays visually uniform around every box regardless of size; corners are clean arcs with no ripples; the dashed border hugs the blob edge exactly; one box yields a rounded rectangle and two boxes a clean elongated blob with no special-case code (rectangle corners always give the hull at least four points); the blob never escapes the gray bounds rectangle.

How the two paths are built

  • Body: raw hull polygon, fill + stroke-width = 2×padding with stroke-linejoin/linecap: round, wrapped in <g opacity> so fill and stroke flatten to one tint. Round-join stroking dilates the hull by exactly the padding with true circular arcs.
  • Border: each hull edge offset outward by the padding along its normal, consecutive offset edges joined by an A (arc) segment of radius = padding around the shared hull vertex. This is the exact outline of the dilated shape, so it can be dashed or solid independently of the body.
  • Bounds: hull min/max extended by padding on every side. Exact, because the dilation extends exactly the padding beyond the hull everywhere.
  • No ClipperLib, no spline, no compensation constant. The only geometry code is a convex hull and one outline builder (~30 lines each).
