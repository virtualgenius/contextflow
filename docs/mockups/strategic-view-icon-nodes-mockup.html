Strategic View: Icon Nodes Instead of Boxes

The Strategic View currently reuses the Flow View's large box nodes. On a Wardley map that fights the medium: boxes dominate the canvas, hide the value chain lines, and read as "containers" rather than positioned components. This mockup follows the visual vocabulary Susanne Kaiser uses in Architecture for Flow: small icon nodes for every element type, with domain classification, build/buy/outsource decisions, and Team Topologies overlays. Example domain is the online school from her InfoQ article (Adaptive Socio-Technical Systems with Architecture for Flow).

New element type. Data storage, search engine, message broker, email, compute platform. Icon varies by kind.

Value chain reads top to bottom: users, needs, processes, bounded contexts, infrastructure components. Evolution reads left to right. The dashed red arrow marks the message broker commoditizing toward a managed service; the black bar marks inertia on the self-managed data storage. Team overlays sit behind the nodes, so flow between teams stays visible.

What Changes vs. Today

Scope notes
  • Contexts shrink from 160px boxes to 52px glyphs in Strategic View only. Flow and Distillation views keep the box nodes.
  • Two new element types: business process/activity and infrastructure component. Both need the full Yjs round-trip (types, schema, sync, mutations) and analytics events.
  • Decision tags and movement annotations (build/buy/outsource, evolution arrow, inertia) are new per-element fields, surfaced as toggleable overlays.
  • Team overlays build on the existing teamId assignment and Team Topologies types; this is the "team boundary overlay" quick win from the Susanne Kaiser feature ideas doc.
  • Icon and color choices reuse the inspector vocabulary (sparkles/wrench/package, amber/blue/slate) so the map and inspector tell one story.
