Strategic View: Icon Nodes Instead of Boxes

The Strategic View currently reuses the Flow View's large box nodes. On a Wardley map that fights the medium: boxes dominate the canvas, hide the value chain lines, and read as "containers" rather than positioned components. This mockup follows the visual vocabulary Susanne Kaiser uses in Architecture for Flow: small icon nodes for every element type, with domain classification, build/buy/outsource decisions, and Team Topologies overlays. The example is the online school from her InfoQ article and Architecture for Flow talk, using her bounded context decomposition.

Try it: drag nodes (edges follow, the evolution band under the node highlights), scroll or use the slider to zoom (decision tags hide first, then labels, mirroring the app's overlay zoom cutoffs), hover or click a node to trace its value chain two hops out, and switch to the dense map to stress-test label crowding. Value chain reads top to bottom: users, needs, bounded contexts, infrastructure components. Bounded contexts are Kaiser's: content creation, class management, course studies, learning support (core); student evaluation, notification handling (supporting); identity & access (generic). Each gets its own stream-aligned team boundary. The dashed red arrow and inertia bar are Wardley-canon movement annotations.

What Changes vs. Today

Scope notes
  • Contexts shrink from 160px boxes to 52px glyphs in Strategic View only. Flow and Distillation views keep the box nodes.
  • Two new element types: capability and infrastructure component. Both need the full Yjs round-trip (types, schema, sync, mutations) and analytics events. Capability is optional in Kaiser's method; needs may connect directly to contexts.
  • Decision tags and movement annotations (build/buy/outsource, evolution arrow, inertia) are new per-element fields, surfaced as toggleable overlays. The movement annotations are Wardley canon rather than Kaiser-specific.
  • Team overlays build on the existing teamId assignment and Team Topologies types, with one stream-aligned boundary per bounded context; this is the "team boundary overlay" quick win from the Susanne Kaiser feature ideas doc.
  • Not drawn yet: Team Topologies interaction modes (collaboration, X-as-a-Service, facilitating) from her canvas. Her as-is to to-be landscape evolution is already covered by ContextFlow's temporal keyframes.
  • Icon and color choices reuse the inspector vocabulary (sparkles/wrench/package, amber/blue/slate) so the map and inspector tell one story.
