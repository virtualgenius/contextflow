Context node badge layout

The header chrome of a context card carries two kinds of information that look the same today but mean very different things. Identity markers (External via dashed border, Legacy, Big Ball of Mud) describe what the context is. Issue markers (info, warning, critical) describe what needs attention. Mixing them in one row reads as a sea of icons. This mockup separates them: identity stays in the header row, attention moves to a per-severity counter at the bottom-right.

Identity
Static facts about the context. Set once during modeling, rarely changes. Reading it does not imply you need to do anything.
Examples: External (dashed border + slate fill), Legacy, Big Ball of Mud.
Attention
Things workshop participants or engineers should look at. The whole point is that they should pull the eye.
Examples: Issue annotations (critical, warning, info). Per-severity counters in the bottom-right.

External: drop the pill, lean on the border

Today external contexts carry both a dashed border and an "EXTERNAL" text pill. The pill overlaps the name on long titles and reads as redundant chrome. The dashed border plus a cool slate fill (distinct from the green/orange ownership colors) is enough to read as "not ours" at any zoom level.

Issues: per-severity counters at the bottom-right

Issues move out of the header entirely and into the bottom-right corner as small counters in the VS Code / Sentry style. Only non-zero severities render; order is critical → warning → info. Each counter keeps its severity color so urgency reads at a glance without flattening the breakdown into a single number.

Rules summary

RuleBehavior
External signalDashed border + slate fill. No "EXTERNAL" text pill. Hover tooltip still names the ownership.
Identity icons — positionTop-right corner, absolutely positioned at top: 4px; right: 4px;. Legacy first, then BBoM with a 4px gap.
NameSits at the top of the card. Gets padding-right only when identity icons are present: 24px for one icon, 44px for two. No margin-top, ever.
Issues — positionBottom-right of the card, absolutely positioned at right: 6px; bottom: 6px;.
Issues — formatPer-severity counters in a single pill: ⊗ N, △ N, ⓘ N. Only non-zero severities render. Order: critical, warning, info.
TooltipsEach counter has a per-severity tooltip ("3 warnings"). Identity icons keep their existing tooltips.

Why per-severity counters

VS Code's Problems tab, Sentry, Datadog, browser dev tools all show separate counts per severity rather than rolling up to a single number. The breakdown matters: "1 critical, 5 warnings" tells a different story from "6 issues, severity unknown." The cost is a few more pixels at the bottom-right; the gain is that workshop facilitators can scan a map and immediately see which contexts have critical issues vs. which just have noise.
