Context node badge layout

Six things can render in the header area of a context card: the External pill (ownership badge), the Legacy icon, the Big Ball of Mud icon, and one or more Issue icons (info, warning, critical). Today they are absolutely positioned with overlapping anchor rules. When more than two appear at once, they collide. This mockup proposes a single explicit header row that lays them out predictably.

Proposed layout

Replace the absolute-positioning rules with a real header row that sits above the name. The row has two zones:

ZoneContentsSizing
Left External pill (only present when ownership === 'external') Auto-width, ellipsises if needed
Right Issue icons (in severity order: critical, warning, info), then Legacy, then Big Ball of Mud Flex row with 2px gap; never wraps; if width is tight, hide overflow icons behind a +N chip

The header row reserves vertical space whether anything renders or not. Empty rows collapse via a fixed min-height. The name and purpose flow naturally below the row with no margin-top overrides anywhere. Name is full-width; ellipsis still applies for very long names.

Edge cases

No badges (the common case)
Header row is empty but still reserves min-height: 18px so cards have a consistent header band. Looks the same as today minus a few pixels at the top.
Shopping Cart
Manages ephemeral and persistent shopping carts across devices.
An empty header is intentional. Cards with and without badges line up vertically when placed side-by-side.

Rules summary

RuleBehavior
Header row is always presentmin-height: 18px, even when empty, so card vertical rhythm stays consistent.
Left zoneExternal pill only. Never icons. Empty for non-external contexts.
Right zone — orderingIssues (critical → warning → info) → Legacy → Big Ball of Mud.
Right zone — overflowIf cluster exceeds available width, collapse trailing items into a +N chip. Keep highest-severity issue visible.
NameFlows naturally below the header row. No margin-top, no padding-right. Full card width minus card padding. Single line, ellipsis on overflow, tooltip on hover.
